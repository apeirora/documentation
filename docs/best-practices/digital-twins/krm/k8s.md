The `kubectl api-resources` command provides a comprehensive list of all the API resources (or <Term>digital twin</Term> types) that Kubernetes with its original intent as container orchestrator supports:
- **Resource Names**: The names of the resources, which are used in kubectl commands to interact with these resources.
- **Shortnames**: Abbreviated names for some resources, which can be used as shortcuts in `kubectl` commands.
- **APIversion**: This column shows the API group and version for each resource. For example, `v1` for core resources or `apps/v1` for resources in the `apps` API group. 
- **Namespaced**: Indicates whether the resource is namespaced (true) or cluster-scoped (false). Namespaced resources exist within a specific namespace, while cluster-scoped resources are not confined to any namespace.
- **Kind**: The kind (or semantic type) of the resource.

In effect, these represent the list of digital twins, or resources, or declarative APIs, and the semantics for immutable infrastructure that modern Kubernetes supports:
